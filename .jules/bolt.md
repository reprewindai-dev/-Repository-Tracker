## 2024-07-20 - Continuous Polling & React Re-renders
**Learning:** The React application utilizes a continuous 4-second polling mechanism in `App.tsx` (`fetchServerState`). This causes the top-level component and its children (like `Dashboard.tsx`) to frequently re-render, even if the actual data hasn't visually changed. If expensive computations (like complex array `.map` or `.reduce` operations, specifically SVG path generation) are placed directly in the component body without memoization, it leads to silent performance drains and tooltip/interaction lag during these re-renders.
**Action:** When working in this specific polling-heavy architecture, always trace data dependencies down the component tree. Aggressively use `useMemo` for derived states, complex math, or array processing inside child components that receive frequently updated props, preventing unnecessary work on every polling tick.
