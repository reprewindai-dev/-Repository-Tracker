## 2025-07-20 - [CRITICAL] Fix Path Traversal SSRF in GitHub URL Parser
**Vulnerability:** The `parseGitHubUrl` function split the requested URL into an `owner` and `repo` using slash splitting but failed to validate the characters in those segments. An attacker could pass a path traversal payload like `https://github.com/../user`, which resolved to `owner: "..", repo: "user"`.
**Learning:** This is critical because the backend immediately passes these parsed components into the GitHub API (`https://api.github.com/repos/${owner}/${repo}`) alongside the server's own GitHub Token (if configured via environment variable). This constitutes a high-impact Server-Side Request Forgery (SSRF). An attacker could hit arbitrary unauthenticated or authenticated GitHub API endpoints (e.g. `https://api.github.com/user` using the internal token).
**Prevention:** Always validate external URL parameters against an explicit allowlist pattern (`/^[a-zA-Z0-9_.-]+$/`) and explicitly deny common traversal payloads (`.` and `..`) before feeding them into internal or outbound HTTP clients.
